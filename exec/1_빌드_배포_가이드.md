## 2. 빌드 시 사용되는 환경 변수

### 2.1 개발 환경 (application-dev.yml)
```bash
# 카카오 OAuth2 설정
SPRING_KAKAO_CLIENT_SECRET=your_kakao_client_secret_here

# 데이터베이스 설정
SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/ssafy-mysql-db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC&characterEncoding=UTF-8&useUnicode=true
SPRING_DATASOURCE_USERNAME=your_db_username
SPRING_DATASOURCE_PASSWORD=your_db_password

# Redis 설정
SPRING_REDIS_HOST=localhost
SPRING_REDIS_PORT=6379

# JWT 설정
JWT_SECRET=your_jwt_secret_key_here
```

### 2.2 프로덕션 환경 (application-prod.yml)
```bash
# 데이터베이스 설정
SPRING_DATASOURCE_URL=jdbc:mysql://your-db-host:3306/your-database
SPRING_DATASOURCE_USERNAME=your-db-username
SPRING_DATASOURCE_PASSWORD=your-db-password

# Redis 설정
SPRING_REDIS_HOST=your-redis-host
SPRING_REDIS_PORT=6379
SPRING_REDIS_PASSWORD=your-redis-password

# 카카오 OAuth2 설정
SPRING_KAKAO_CLIENT_ID=your_kakao_client_id
SPRING_KAKAO_CLIENT_SECRET=your_kakao_client_secret
SPRING_KAKAO_REDIRECT_URI=https://api.promisenow.store/oauth2/authorization/kakao

# JWT 설정
SPRING_JWT_TOKEN=your-jwt-secret-key

# 프론트엔드 리다이렉트 URI
SPRING_FRONT_REDIRECT_URI=https://promisenow.store
```

### 2.3 WebRTC 서버 환경 변수
```bash
# 도메인 설정
DOMAIN=webrtc.promisenow.store

# HTTPS 인증서 경로
HTTPS_CERT_FULLCHAIN=/etc/letsencrypt/live/webrtc.promisenow.store/fullchain.pem
HTTPS_CERT_PRIVKEY=/etc/letsencrypt/live/webrtc.promisenow.store/privkey.pem

# mediasoup 설정
MEDIASOUP_LISTEN_IP=0.0.0.0
MEDIASOUP_ANNOUNCED_IP=your-server-public-ip
MEDIASOUP_MIN_PORT=40000
MEDIASOUP_MAX_PORT=49999

# Protoo 서버 포트
PROTOO_LISTEN_PORT=4443
```

## 3. 빌드 및 배포 과정

### 3.1 사전 요구사항
```bash
# 필수 소프트웨어 설치
- Docker & Docker Compose
- Java 21
- Node.js 20+
- pnpm
- Git
```

### 3.2 백엔드 빌드
```bash
cd backend-spring

# 테스트 파일 제거 (빌드 오류 방지)
rm -rf src/test

# Gradle 빌드
./gradlew clean build -x test

# JAR 파일 생성 확인
ls -la build/libs/
```

### 3.3 프론트엔드 빌드
```bash
cd frontend

# 의존성 설치
pnpm install

# 프로덕션 빌드
pnpm run build

# 빌드 결과 확인
ls -la dist/
```

### 3.4 WebRTC 서버 설정
```bash
cd backend-webrtc

# 설정 파일 복사
cp config.prod.js config.js

# 의존성 설치
npm install

# SSL 인증서 확인
ls -la certs/
```

## 4. 배포 시 특이사항

### 4.1 Docker Compose 배포
```bash
# 프로덕션 환경 변수 설정
export SPRING_PROFILES_ACTIVE=prod

# Docker Compose 실행
docker-compose up -d

# 서비스 상태 확인
docker-compose ps
```

### 4.2 수동 배포
```bash
# 백엔드 실행
java -jar -Dspring.profiles.active=prod backend-spring/build/libs/api-0.0.1-SNAPSHOT.jar

# WebRTC 서버 실행
cd backend-webrtc
node server.js

# 프론트엔드 서빙 (Nginx 사용 권장)
```

### 4.3 SSL 인증서 설정
- Let's Encrypt 인증서 사용
- 도메인별 인증서 필요:
  - `promisenow.store` (프론트엔드)
  - `api.promisenow.store` (백엔드 API)
  - `webrtc.promisenow.store` (WebRTC 서버)

### 4.4 방화벽 설정
```bash
# 필요한 포트 개방
- 80 (HTTP)
- 443 (HTTPS)
- 8080 (Spring Boot)
- 4443 (WebRTC)
- 40000-49999 (WebRTC 미디어)
- 3306 (MySQL)
- 6379 (Redis)
```

## 5. DB 접속 정보 및 프로퍼티 파일

### 5.1 주요 프로퍼티 파일 목록
- `backend-spring/src/main/resources/application-dev.yml` (개발 환경)
- `backend-spring/src/main/resources/application-prod.yml` (프로덕션 환경)
- `backend-spring/src/main/resources/application-test.yml` (테스트 환경)
- `backend-spring/src/main/resources/schema.sql` (DB 스키마)
- `docker-compose.local.yml` (로컬 개발용)
- `docker-compose.yml` (프로덕션용)

### 5.2 데이터베이스 접속 정보
```yaml
# 개발 환경
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ssafy-mysql-db
    username: ${SPRING_DATASOURCE_USERNAME}
    password: ${SPRING_DATASOURCE_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver

# Redis 설정
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password: 
      database: 0
```

### 5.3 ERD 주요 테이블
- `users`: 사용자 정보
- `room`: 방 정보
- `room_user`: 방 참여자 정보
- `availability`: 일정 가능 시간
- `chat_message`: 채팅 메시지
- `leaderboard`: 리더보드 정보
