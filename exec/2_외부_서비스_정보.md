# PromiseNow 외부 서비스 정보

## 1. 카카오 OAuth2 (소셜 인증)

### 1.1 서비스 개요
- **서비스명**: 카카오 로그인 (OAuth2)
- **용도**: 사용자 인증 및 로그인
- **설정 위치**: `backend-spring/src/main/resources/application-dev.yml`, `application-prod.yml`

### 1.2 가입 및 설정 방법

#### 1.2.1 카카오 개발자 계정 생성
1. [Kakao Developers](https://developers.kakao.com) 접속
2. 카카오 계정으로 로그인
3. 애플리케이션 생성

#### 1.2.2 애플리케이션 설정
```yaml
# 개발 환경 설정
spring:
  security:
    oauth2:
      client:
        registration:
          kakao:
            client-id: ${SPRING_KAKAO_CLIENT_ID}  # 카카오 앱 키
            client-secret: ${SPRING_KAKAO_CLIENT_SECRET}  # 카카오 시크릿 키
            redirect-uri: "{baseUrl}/login/oauth2/code/{registrationId}"
            authorization-grant-type: authorization_code
            client-authentication-method: client_secret_post
        provider:
          kakao:
            authorization-uri: https://kauth.kakao.com/oauth/authorize
            token-uri: https://kauth.kakao.com/oauth/token
            user-info-uri: https://kapi.kakao.com/v2/user/me
            user-name-attribute: id
```

#### 1.2.3 필수 설정 항목
- **플랫폼 등록**: Web 플랫폼 추가
- **Redirect URI**: 
  - 개발: `http://localhost:8080/login/oauth2/code/kakao`
  - 프로덕션: `https://api.promisenow.store/login/oauth2/code/kakao`
- **동의항목**: 
  - 닉네임 (profile_nickname)
  - 프로필 사진 (profile_image)
  - 이메일 (account_email)

#### 1.2.4 환경 변수 설정
```bash
# 개발 환경
export SPRING_KAKAO_CLIENT_SECRET="your_kakao_client_secret_here"

# 프로덕션 환경
export SPRING_KAKAO_CLIENT_ID="your_kakao_client_id"
export SPRING_KAKAO_CLIENT_SECRET="your_kakao_client_secret"
export SPRING_KAKAO_REDIRECT_URI="https://api.promisenow.store/oauth2/authorization/kakao"
```

### 1.3 API 사용 정보
- **API 버전**: v2
- **사용자 정보 엔드포인트**: `https://kapi.kakao.com/v2/user/me`
- **토큰 갱신**: 자동 처리 (Spring Security OAuth2)
- **에러 처리**: `OAuth2LoginSuccessHandler`, `OAuth2LoginFailureHandler`

## 2. mediasoup (WebRTC 서버)

### 2.1 서비스 개요
- **서비스명**: mediasoup
- **용도**: WebRTC 미디어 서버 (화상회의, 음성통화)
- **설정 위치**: `backend-webrtc/config.js`, `config.prod.js`

### 2.2 설치 및 설정

#### 2.2.1 의존성 설치
```bash
cd backend-webrtc
npm install mediasoup@3.x
npm install protoo-server@4.x
npm install express body-parser
```

#### 2.2.2 환경 변수 설정
```bash
# 도메인 설정
export DOMAIN="webrtc.promisenow.store"

# HTTPS 인증서 경로
export HTTPS_CERT_FULLCHAIN="/etc/letsencrypt/live/webrtc.promisenow.store/fullchain.pem"
export HTTPS_CERT_PRIVKEY="/etc/letsencrypt/live/webrtc.promisenow.store/privkey.pem"

# mediasoup 설정
export MEDIASOUP_LISTEN_IP="0.0.0.0"
export MEDIASOUP_ANNOUNCED_IP="your-server-public-ip"
export MEDIASOUP_MIN_PORT="40000"
export MEDIASOUP_MAX_PORT="49999"

# Protoo 서버 포트
export PROTOO_LISTEN_PORT="4443"
```

#### 2.2.3 SSL 인증서 설정
```bash
# Let's Encrypt 인증서 발급
sudo certbot certonly --standalone -d webrtc.promisenow.store

# 인증서 권한 설정
sudo chmod 644 /etc/letsencrypt/live/webrtc.promisenow.store/fullchain.pem
sudo chmod 600 /etc/letsencrypt/live/webrtc.promisenow.store/privkey.pem
```

## 3. Redis (캐시 및 세션 저장소)

### 3.1 서비스 개요
- **서비스명**: Redis
- **용도**: JWT 리프레시 토큰 저장, 세션 캐시, 실시간 데이터 저장
- **설정 위치**: `docker-compose.local.yml`, `application-dev.yml`, `application-prod.yml`

### 3.2 설치 및 설정

#### 3.2.1 Docker를 통한 설치
```yaml
# docker-compose.local.yml
redis:
  image: redis:7.2-alpine
  container_name: redis-local
  restart: always
  ports:
    - "6379:6379"
  volumes:
    - redis_data:/data
  command: redis-server --appendonly yes --requirepass ""
  healthcheck:
    test: ["CMD", "redis-cli", "ping"]
    interval: 5s
    timeout: 3s
    retries: 5
```

#### 3.2.2 Spring Boot 설정
```yaml
spring:
  data:
    redis:
      host: ${SPRING_REDIS_HOST:localhost}
      port: ${SPRING_REDIS_PORT:6379}
      password: ${SPRING_REDIS_PASSWORD:}
      database: 0
      timeout: 2s
      lettuce:
        pool:
          max-active: 8
          max-idle: 8
          min-idle: 0
          max-wait: -1ms
```

### 3.3 사용 목적
- **JWT 리프레시 토큰 저장**: `RefreshTokenService`
- **실시간 채팅 메시지 캐시**: WebSocket 메시지 임시 저장
- **사용자 세션 관리**: 로그인 상태 관리
- **리더보드 데이터**: 실시간 순위 정보

## 4. MySQL (데이터베이스)

### 4.1 서비스 개요
- **서비스명**: MySQL 8.0
- **용도**: 메인 데이터베이스 (사용자, 방, 일정, 채팅 등)
- **설정 위치**: `docker-compose.local.yml`, `application-dev.yml`

### 4.2 설치 및 설정

#### 4.2.1 Docker를 통한 설치
```yaml
# docker-compose.local.yml
mysql:
  image: mysql:8.0
  container_name: mysql-local
  restart: always
  ports:
    - "3306:3306"
  environment:
    MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
    MYSQL_USER: ${MYSQL_USER}
    MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    MYSQL_DATABASE: ${MYSQL_DATABASE}
  command:
    - "--character-set-server=utf8mb4"
    - "--collation-server=utf8mb4_unicode_ci"
    - "--default-time-zone=UTC"
    - "--sql-mode=STRICT_TRANS_TABLES,NO_ZERO_DATE,NO_ZERO_IN_DATE,ERROR_FOR_DIVISION_BY_ZERO"
```

#### 4.2.2 Spring Boot 설정
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ssafy-mysql-db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC&characterEncoding=UTF-8&useUnicode=true
    username: ${SPRING_DATASOURCE_USERNAME}
    password: ${SPRING_DATASOURCE_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
        format_sql: true
        connection:
          characterEncoding: utf8
          useUnicode: true
```

### 4.3 주요 테이블 구조
- **users**: 사용자 기본 정보
- **room**: 방 정보
- **room_user**: 방 참여자 정보
- **availability**: 일정 가능 시간
- **chat_message**: 채팅 메시지
- **leaderboard**: 리더보드 정보

## 5. Slack Webhook (모니터링)

### 5.1 서비스 개요
- **서비스명**: Slack Webhook
- **용도**: 에러 알림 및 모니터링
- **설정 위치**: `application-dev.yml`, `application-prod.yml`

### 5.2 설정 방법

#### 5.2.1 Slack Webhook URL 생성
1. Slack 워크스페이스 접속
2. 앱 추가 → "Incoming Webhooks" 검색
3. 채널 선택 및 Webhook URL 생성

#### 5.2.2 Spring Boot 설정
```yaml
webhook:
  slack:
    url: ${SLACK_WEBHOOK_URL}
  enable:
    handled-exceptions: true
    unhandled-exceptions: true
```

### 5.3 알림 대상
- **핸들링된 예외**: 비즈니스 로직 오류
- **핸들링되지 않은 예외**: 시스템 오류
- **로그 레벨**: ERROR, WARN

## 6. Let's Encrypt (SSL 인증서)

### 6.1 서비스 개요
- **서비스명**: Let's Encrypt
- **용도**: 무료 SSL 인증서 발급
- **도메인**: promisenow.store, api.promisenow.store, webrtc.promisenow.store

### 6.2 인증서 발급 및 갱신

#### 6.2.1 certbot 설치
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install certbot

# CentOS/RHEL
sudo yum install certbot
```

#### 6.2.2 인증서 발급
```bash
# 도메인별 인증서 발급
sudo certbot certonly --standalone -d promisenow.store
sudo certbot certonly --standalone -d api.promisenow.store
sudo certbot certonly --standalone -d webrtc.promisenow.store
```

#### 6.2.3 자동 갱신 설정
```bash
# crontab에 갱신 작업 추가
sudo crontab -e

# 매일 자정에 갱신 시도
0 0 * * * /usr/bin/certbot renew --quiet
```
